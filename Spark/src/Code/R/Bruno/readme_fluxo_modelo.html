README: ordem de execução do modelo booleano com redes reais

Bloco 0 — organização dos arquivos
Arquivos novos (criados nesta sessão)
identify_core_periphery.R Classifica espécies como core/periférica pelo grau
ecosystem_services_network_v2.R Cria rede de serviços por core / periphery / random
calcular_metricas_rede.R Calcula NODF, Q, conectância, grau médio por rede
explore_parameters_v3_redes_reais.R Loop principal adaptado para redes reais
run_model.R Script master — orquestra tudo e salva resultados
Arquivos originais que devem estar na mesma pasta
functions_Bruno.R environment(), alpha_fun(), Cf()
model_node_actualization.R simulation() — núcleo do modelo booleano
eco_services_iteraction.R ecosystem() — calcula serviços ao longo do tempo
Bloco 1 — antes de rodar (uma única vez)
0
Instalar pacotes necessários
Só precisa rodar uma vez na máquina.
install.packages("bipartite") install.packages("dplyr") install.packages("readr")
1
Definir o caminho das redes no run_model.R
Linha a editar no script master:
caminho_redes <- "seu/caminho/aqui"
Bloco 2 — ordem de execução dentro do run_model.R
1
Carregar todas as funções
7 source() na ordem exata abaixo:
source("functions_Bruno.R") source("model_node_actualization.R") source("eco_services_iteraction.R") source("identify_core_periphery.R") source("ecosystem_services_network_v2.R") source("calcular_metricas_rede.R") source("explore_parameters_v3_redes_reais.R")
2
Ler as redes reais do disco
Lista todos os CSVs recursivamente, lê cada um com importar_rede_bipartida(), retorna lista nomeada de matrizes bipartidas binárias.
3
Calcular métricas estruturais
Chama calcular_metricas_todas() para todas as redes de uma vez. Salva automaticamente em disco.
conectância grau médio NODF (aninhamento) Q (modularidade) n plantas / animais
⚠ NODF e Q são lentos para redes grandes. Para redes com >200 espécies, considere rodar com calc_nodf=FALSE e calcular separadamente.
4
Rodar o modelo booleano
Chama explore_parameters_v3_redes_reais(). Para cada rede, varia o provedor de serviços e roda n_replicates vezes. Internamente, em cada simulação:
4a
bipartida → quadrada
Conversão interna no loop
4b
identify_core_periphery_bipartite()
Classifica espécies pelo grau
4c
ecosystem_services_network_v2()
Cria rede ES para core / periphery / random
4d
environment() → simulation()
Cria θ(t) e roda dinâmica booleana
4e
ecosystem()
Calcula serviços ao longo do tempo
5
Salvar resultados
Gerado automaticamente ao final do run_model.R:
metricas_redes.csv metricas_redes.rds resultados_modelo.rds resultados_modelo.csv
Bloco 3 — o que está no dataframe de saída
Identificação
sim_id, nome_rede, service_providers, replicate
Estrutura da rede
conectância, NODF, Q, grau médio, n espécies
Core / periphery
n_core, n_periphery, persistência por grupo
Dinâmica
persistence_species, persistence_interactions, convergência
Serviços
serv_initial, serv_final, loss, riqueza_servicos
Históricos (listas)
prop_species, prop_interactions, services ao longo do tempo
